FordGoBike System Data Exploratation¶

FordGoBike (later BayWheels) is a Bay Area bike sharing system. In this project, the trip data of FordGoBike from 2018 to 2019 will be explored.

Data Gathering¶

The data is available on 'https://s3.amazonaws.com/baywheels-data/index.html'. This data will be downloaded programmatically using the Selenium and Beautiful Soup packages and then read to a dataframe using requests library.

Structure of the dataset:¶

The final cleaned dataset has 4370704 trip data with 18 features

  1. duration_sec (int)
  2. start_time (datetime)
  3. end_time (datetime)
  4. start_station_id (float)
  5. start_station_name (string)
  6. start_station_latitude (float)
  7. start_station_longitude (float)
  8. end_station_id (float)
  9. end_station_name (string)
  10. end_station_latitude (float)
  11. end_station_longitude (float)
  12. bike_id (int)
  13. user_type (string)
  14. bike_share_for_all_trip (string)
  15. rental_access_method (string)
  16. year (int)
  17. month (int)
  18. weekday (int)

The bike_share_for_all_trip and rental_access_method feature data is available only for certain months.

Main feature of interest in the dataset:¶

I am interested in analysing the influence of various features on the demand for bikes (i.e number of bike trips) and the duration of the trip.

Features expected to affect the main feature¶

I expect the weekday, month, year of the trip as well as user type to have strongest effect on the number of trips. The effect of bike_share_for_all_trip and rental_access_method would also be interesting to study
